# 2026-03-09 - Infographic Prompt Generator 8-Part Refactoring

## 工作目标
优化信息图提示生成器模板，实现 8 部分并行推理和合并架构。

## 完成内容

### 1. 模板优化（已完成）
- 删除冗余的 "Frosted Glass"（Medium 和 Material 中 94% 重复）
- 删除 2 个重复的 Atmosphere Setting 选项
- 删除 2 个重复的 Avoid 列表项
- 从所有 9 个 Image Quality 行中删除 "8K"
- 删除整个 Style Reference Library 表格（10 行）
- **结果**: 11804 → 8990 字符，~2951 → ~2248 tokens 输入（节省 ~703 tokens，~984 round-trip）

### 2. JSON 字段标准化
- 统一所有 JSON 字段名为下划线命名（7 个必需顶层字段）
- 更新 `_validate_json_structure` 使用下划线命名
- 修复嵌套字段命名（Core_Theme, Canvas_Ratio, Language_Standards 等）

### 3. 8 部分并行架构
创建了 8 个独立的 operator，每个生成一个部分：
- `part1_config.py` - Infographic_Global_Configuration
- `part2_visual.py` - Visual_Style
- `part3_headline.py` - Headline_Art_System
- `part4_label.py` - Content_Label_System
- `part5_nodes.py` - Core_Content_Elements_Nodes (4-8 个节点)
- `part6_interaction.py` - Interaction_Features (可选)
- `part7_constraints.py` - Hard_Constraints (Must_Achieve + Avoid)
- `part8_rendering.py` - Rendering_Parameters_and_Negatives

### 4. 基类重构
- 创建 `InfographicPartBaseGenerator` 基类（不注册到注册表）
- 所有 8 个 operator 继承自基类（代码减少 ~64%）
- 基类提供通用方法：
  - `_clean_json_output`: 清理 Markdown 代码块标记
  - `_format_prompt`: 格式化 prompt 模板
  - `run`: 执行生成、解析、验证、保存

### 5. 参考值集成
- 从 `/tmp/flowus_json/` 读取 375 个真实 JSON 文件
- 为每个 part 添加参考值表格
- 创建 10 种主题类型的风格参考表：
  - 设计/艺术、个人成长、少女风、科技/数据、经济/商业
  - 生活/美食、音乐/活动、教育/知识、心理/情感、建筑/空间

### 6. 创意自由提醒
为每个 prompt 添加重要提醒：
- **联想扩展**: 组合不同参考值中的关键词
- **风格融合**: 将 style_preference 与主题融合
- **创意发挥**: 发明新的描述性词汇
- **保持连贯**: 确保所有字段协调一致
- **参考前序输出**: 使用 previous_output 保持一致性

### 7. 输入参数增强
每个 part 的 prompt 现在包含：
- `{topic}` - 主题
- `{content_description}` - 内容描述
- `{style_preference}` - 风格偏好
- `{previous_output}` - 前序 part 输出（part2-part8）

### 8. 合并器
- 创建 `InfographicPromptMerger` 类
- 合并 8 个部分的 JSON 输出为单个完整对象
- 自动删除已合并的输入字段

## 技术要点
- 所有输出为纯 JSON，无 Markdown
- 验证要求：Core_Content_Elements_Nodes 4-8 个节点，Must_Achieve/Avoid 至少 5 项
- 基类 `run` 方法参数：`input_topic_key`, `input_content_key`, `input_previous_output_keys`, `output_key`
- 使用 `json.dumps(prev_data, ensure_ascii=False, indent=2)` 格式化前序输出
