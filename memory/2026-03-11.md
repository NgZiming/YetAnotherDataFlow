# 2026-03-11 - Infographic Prompt 系统重构

## 主要工作

### 1. Prompt 问题分析与改进（与 ViGenEval 对比）

**发现的问题：**
- 字段语义混淆（Visual_Anchor_Description 和 Headline/Description 用途边界不清）
- 负向提示词分散且不一致
- 约束表达不够结构化
- 角色定义模糊
- 输出格式约束不足

**改进措施：**
- 强化角色定义（中文 + 核心职责清单）
- 添加字段用途说明（⚠️ 标记每个字段的精确用途）
- 添加必须遵守的约束（4-5 条强制约束）
- 添加回避元素（明确列出不应做的事情）
- 添加输出前自检（引导 LLM 自我验证）

### 2. 前序输出字段映射优化

**问题：** 前序输出被总结为一个字段，LLM 不知道哪个字段对应哪个 Part

**解决：**
- 为每个 Part 添加前序输出字段说明表格
- 使用实际 JSON Key（如 `Infographic_Global_Configuration` 而不是 `part1_config`）
- 添加引用建议（如"需要参考布局时，查看 `Infographic_Global_Configuration.Layout_Architecture`"）
- 每个 Part 只列出真正需要的前序 Part，精简 prompt

### 3. Part 合并（7 → 4 个请求）

**合并方案：**
| 新 Part | 包含原 Part | OUTPUT_KEY |
|--------|-----------|------------|
| part12_config_style | Part 1 + Part 2 | `part12_config_style` |
| part34_text_system | Part 3 + Part 4 | `part34_text_system` |
| part5_nodes | Part 5 | `part5_nodes` |
| part78_final_constraints | Part 7 + Part 8 | `part78_final_constraints` |

**效果：**
- 减少 3 个 API 请求（减少 43%）
- Part 5 (Nodes) 保持独立，因为内容生成最关键
- 合并的 Part 让 LLM 可以协同设计相关元素

### 4. 动态节点数量逻辑

**基于 style_selector 关键词的动态决策规则：**

| 风格类型 | Visual_Aesthetics 关键词特征 | 推荐节点数 |
|---------|---------------------------|-----------|
| 极简/抽象风格 | 极简主义 / 格式塔 / 包豪斯 / 弥散光感 / Notion 极简 | 3-5 个 |
| 教育/知识风格 | 教科书风格 / 清晰图解 / 知识导图 / 黑板手绘 | 4-6 个 |
| 生活/温暖风格 | 织物拼贴 / 手账风格 / 童话绘本 / 水墨画 | 4-6 个 |
| 科技/数据风格 | 3D 等距 / 数据可视化 / 赛博朋克 / 神经网络 | 5-7 个 |
| 极繁/装饰风格 | 极繁主义 / 波普艺术 / 孟菲斯风格 / 洛可可华丽 | 6-8 个 |
| 内容驱动 | 内容描述包含明确的分类/步骤/列表 | 跟随内容数量 |
