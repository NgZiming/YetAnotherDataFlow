## OpenClaw CLI Serving 实现

### 背景
放弃了之前基于 WebSocket API 的实现思路（`api_openclaw_serving.py`），改用 openclaw-flow 的 CLI 方式。

### 主要工作
1. **删除旧代码**
   - 删除 `api_openclaw_serving.py`（WebSocket API 版本）
   - 删除 `test_api_openclaw_serving.py`

2. **创建 `cli_openclaw_serving.py`**
   - 通过 `openclaw agent` CLI 命令执行任务
   - 每个请求在独立的 session 中运行
   - 使用 `ThreadPoolExecutor` 并发处理
   - 对象可 pickle 拷贝（适合分布式任务）

3. **代码整理**
   - 添加详细注释和日志
   - 清理未使用的函数和参数
   - 修复 `_execute_single_query` 的执行状态检查

4. **更新 `__init__.py`**
   - 导出 `CLIOpenClawServing` 和 `create_openclaw_serving`

5. **提交和 Push**
   - dataflow 子模块：`251c986` - Add CLIOpenClawServing
   - 成功 push 到 `gitlab.sh.sensetime.com:wuziming/dataflow.git`

### 设计要点
- **不依赖 WebSocket API** - 直接用 CLI，更稳定
- **独立 session** - 每个请求有自己的 session_id
- **并发支持** - 每次 `generate_from_input` 创建临时线程池
- **可 pickle** - 适合 multiprocessing 场景
